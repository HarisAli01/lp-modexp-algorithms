The Simplex Method is the most famous algorithm for solving Linear Programming problems. It iteratively moves from one vertex of the feasible region to an adjacent vertex with a better objective value.

Key Idea: Start at a corner of the feasible region. Keep moving to neighboring corners that improve the objective until no improvement is possible.

Pivot Step: Select entering variable (most negative in row), leaving variable (smallest ratio test), then perform Gaussian elimination.
Feasible Region: The set of all points satisfying all constraints. It forms a convex polytope.

Optimal Solution: Always occurs at a vertex of the feasible region (if it exists).

Bland's Rule: Choose the smallest-indexed entering/leaving variable to avoid cycling.

Tableau: Matrix form showing coefficients and current values. Each column is a variable.
Engineering/Business Case: Production planning for two products x1 and x2.
Input: objective coefficients c, resource matrix A, capacity vector b.
Output: optimal production quantities x and optimal profit z.
Objective: maximize z = c^T x subject to Ax ≤ b, x ≥ 0.
Why Simplex: LP model is linear with exact constraints, and optimality is required.
When not applicable: nonlinear objectives/constraints, integer-only decisions (need MILP/IP), uncertain data (need stochastic/robust variants).

Linear Programming: The Simplex Method
Lecture Notes - Simplex Method
Learning Roadmap
1What Is Linear Programming?

Linear Programming (LP) optimizes a linear objective under linear constraints. Geometrically, constraints create a feasible region, and the best value is attained at a vertex if an optimum exists.

Canonical Form: maximize z = c^T x, subject to A x ≤ b and x ≥ 0.
2Standard Form and Tableau

Each inequality constraint gets a slack variable so we can write equalities. The tableau stores all coefficients and current RHS values, making each pivot an elementary row-operation step.

Example: 2x + y ≤ 4 becomes 2x + y + s1 = 4, where s1 ≥ 0.
3Simplex Core Loop

Simplex moves from one basic feasible solution to an adjacent one that improves the objective. In tableau form, each move is a pivot.

repeat choose entering variable (improves objective) choose leaving variable (minimum ratio test) pivot to update basis until no entering variable exists (optimal)
The ratio test keeps the next solution feasible while improving objective value.
4Pivot Rules and Cycling

The entering variable is chosen to improve objective fastest (largest coefficient rule) or safely (Bland's rule). Bland's rule avoids cycling in degenerate LPs.

Practical Tip: Use Bland's rule when ties and degeneracy appear repeatedly.
5Worked Walkthrough
For max z = 3x + 2y with 2x + y ≤ 4 and x + 2y ≤ 4, start from an initial basic feasible corner, pivot to improve z, and stop when all reduced costs indicate no improvement.
6Edge Cases

LPs may be unbounded (objective can grow forever), infeasible (no point satisfies all constraints), or degenerate (a pivot does not change objective).

The simulator examples already include normal, unbounded, infeasible, and degenerate scenarios to practice diagnosis.
7When To Use Simplex

Use Simplex when the model is linear and exact optimality is required: production planning, transportation, assignment, diet/blending, scheduling, and resource allocation.

Simplex is also strong for sensitivity analysis, shadow prices, and understanding trade-offs via dual variables.

The Fast Fourier Transform (FFT) computes the Discrete Fourier Transform in O(n log n) time instead of O(n²).

Key Idea: Recursively divide the problem: if n is even, split into even and odd indices, compute smaller FFTs, then combine.

Cooley-Tukey Algorithm: The most popular FFT algorithm. Assumes n is a power of 2.
DFT Matrix: Fn[j,k] = ω^(jk) where ω = e^(-2πi/n).

Twiddle Factor: ω^k multipliers at each stage.

Butterfly: Basic operation combining two values with a twiddle factor.

Frequency Domain: Output shows magnitude and phase of frequency components.
Problem Identification: Spectrum analysis / polynomial multiplication where direct DFT is too slow.
Recurrence: T(n) = 2T(n/2) + O(n), so T(n) = O(n log n).
Naive Baseline: O(n^2) complex operations.
Why faster: Recursive reuse of even/odd subproblems and butterfly combining avoids repeated work.
When worth using: medium to large n, repeated transforms, or real-time signal tasks.
Practical caveats: constants, cache behavior, and small-n overhead can reduce theoretical speedup.
FFT(x, n):
if n = 1: return x
split x into even (x_even) and odd (x_odd)
y_even ← FFT(x_even)
y_odd ← FFT(x_odd)
for k ← 0 to n/2 - 1:
w ← e^(-2πik/n)
y[k] ← y_even[k] + w·y_odd[k]
y[k+n/2] ← y_even[k] - w·y_odd[k]
return y

FFT: The Discrete Fourier Transform
Lecture Notes - Fast Fourier Transform
Learning Roadmap
1What DFT Computes

The Discrete Fourier Transform maps samples from time domain to frequency domain. Each output coefficient measures how much of a specific sinusoidal frequency exists in the input signal.

DFT: X[k] = Sum over j of x[j] * e^(-2pi i j k / n)
2Why Naive DFT Is Slow

Directly computing every X[k] requires n multiplications for each of n outputs, producing O(n^2) complexity. This becomes expensive for long signals.

3Divide and Conquer Idea

For n as power of 2, split x into even-index and odd-index sequences, recursively solve both subproblems, then merge with twiddle factors using butterfly operations.

FFT(x): if n = 1 return x E = FFT(x_even) O = FFT(x_odd) for k = 0..n/2-1: X[k] = E[k] + w_k * O[k] X[k+n/2] = E[k] - w_k * O[k]
4Butterfly and Twiddle

A butterfly combines two sub-results using a root of unity. Twiddle factors rotate the odd branch in the complex plane before add/subtract combination.

For each k: X[k] = E[k] + w_k O[k], and X[k+n/2] = E[k] - w_k O[k].
5Complexity
Naive DFT uses O(n^2) operations. FFT uses O(n log n), which is dramatically faster for large n.
At n = 1024, n^2 is over one million operations scale, while n log2 n is around ten thousand scale.
6Inverse FFT and Convolution

The inverse FFT reconstructs time-domain signals from frequency coefficients. FFT is also used for fast convolution by multiplying spectra and applying inverse transform.

7Applications

Use FFT for spectrum analysis, filtering, denoising, image processing, OFDM communication, audio equalization, and fast polynomial multiplication via convolution.

Always check sampling assumptions and signal length padding to avoid interpretation mistakes.

Modular Exponentiation (Square-and-Multiply) computes a^b mod n efficiently in O(log b) multiplications.

Key Idea: Use the binary representation of b. Scan bits and square the accumulator at each step. If the bit is 1, also multiply by a.

Critical for RSA: Computing RSA encryption/decryption (where exponents are huge) would be impossible without this algorithm.
Binary Exponentiation: Express b in binary. Example: 13 = 1101₂ = 1·8 + 1·4 + 0·2 + 1·1.

Property: a^(2b) = (a^b)².

Modular Arithmetic: (a·b) mod n = ((a mod n) · (b mod n)) mod n.

Left-to-Right: Start from the highest bit.

Right-to-Left: Start from the lowest bit.
Pipeline: key generation (p, q, n, φ, e, d) → encryption c = m^e mod n → decryption m = c^d mod n.
Number Theory Used: modular arithmetic, gcd(e, φ) = 1, modular inverse d = e^(-1) mod φ, Euler theorem basis for correctness.
Correctness Check: verify decrypted message equals original message for multiple inputs.
Security Assumption: practical RSA security relies on hardness of factoring n = pq for large primes.
SQUARE-AND-MULTIPLY(a, b, n):
result ← 1
binary ← binary representation of b (high to low)
for each bit in binary:
result ← (result * result) mod n
if bit == 1:
result ← (result * a) mod n
return result

Modular Exponentiation: Square-and-Multiply
Lecture Notes - Modular Exponentiation
Learning Roadmap
1Why Direct a^b Is Hard

Naively computing a^b requires b multiplications and produces huge intermediate integers. With large b (e.g., 2048-bit exponents), this is impractical.

Core Rule: reduce modulo n after every multiplication to keep numbers bounded.
2Binary Exponent Insight

Exponent b is represented in binary. Each bit controls whether a specific power of a contributes to the final product. This turns repeated multiplication into a logarithmic bit-scan process.

3Left-to-Right Method

Scan bits from most significant to least significant. Square every step, multiply by a only when the current bit is 1.

result = 1 for each bit in binary(b): result = (result * result) mod n if bit == 1: result = (result * a) mod n
4Right-to-Left Method

Scan bits from least significant to most significant using a running base power. If a bit is 1, multiply it into result; then square the base each round.

5Worked Example
Compute 3^13 mod 100. Binary(13)=1101. Process each bit with square always, multiply on 1 bits. Final value = 23.
6Complexity and Correctness

The algorithm performs O(log b) modular multiplications. Correctness follows from binary decomposition of b and modular multiplication properties.

(x * y) mod n = ((x mod n) * (y mod n)) mod n allows safe reduction at each step.
7Cryptography Uses

Use for RSA encryption/decryption, digital signatures, Diffie-Hellman key exchange, Miller-Rabin primality tests, and any modular power in number theory.

Time complexity O(log b) is exactly why real-world public-key cryptography is feasible at large key sizes.
